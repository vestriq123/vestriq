# Vestriq Production Deployment Guide

This guide details the step-by-step procedure to provision production database resources, configure media management services, and deploy the Vestriq Next.js application on Vercel.

---

## 1. Database Provisioning (Supabase)

Supabase provides a hosted PostgreSQL instance with connection pooling. Because Vercel serverless functions scale dynamically, we use **PgBouncer** (transaction pooling) for API routes and a **Direct Connection** (session pooling) for running database migrations and seeds.

### Step-by-Step Setup:
1. Log in to [Supabase Console](https://database.new) and create a new project.
2. Store your Database Password securely.
3. Once the project is provisioned, navigate to **Project Settings** > **Database** > **Connection Strings**.
4. Capture the following details:
   - **Transaction Connection Pooler URL** (typically port `6543` with `?pgbouncer=true`): This will be your `DATABASE_URL`. Ensure you append `&connection_limit=1` to the URL query parameters to avoid exhausting PostgreSQL connections.
   - **Direct Connection String** (typically port `5432` or direct pooler configuration): This will be your `DIRECT_URL` used for migrations.

## 2. Media Asset Management (Cloudinary)

Vestriq allows users to upload screenshots of bank transfers/cryptocurrency transactions as proof of payment. We store these securely on Cloudinary.

### Step-by-Step Setup:
1. Create a free account at [Cloudinary](https://cloudinary.com).
2. Open the **Console Dashboard** and retrieve:
   - **Cloud Name**
   - **API Key**
   - **API Secret**
3. Navigate to **Settings** > **Upload** and ensure an **unsigned upload preset** is configured if client-side browser direct uploads are utilized, or manage signed uploads securely through API keys.

## 3. Local Preparation & Migrations

Before pushing the final code, initialize the production database tables and seed metadata:

```bash
# 1. Clone/navigate to the codebase root and create your production .env config
cp .env.example .env

# Configure DATABASE_URL and DIRECT_URL inside the newly created .env file.

# 2. Push the Prisma database schema into Supabase PostgreSQL
npx prisma db push

# 3. Seed initial system parameters (Create Default Admin, default plans, default roles)
npm run seed
```

*Note: The default administrator credentials generated by the seed script will be printed to your console.*

---

## 4. Deploying to Vercel

1. Log in to the [Vercel Dashboard](https://vercel.com) and click **Add New** > **Project**.
2. Connect your Git repository (GitHub / GitLab / Bitbucket).
3. Under **Configure Project**:
   - **Framework Preset**: Next.js
   - **Root Directory**: `./` (or directory of your repo)
   - **Build Command**: `npm run build` (Which automatically executes `prisma generate && next build`)
4. Add the following **Environment Variables**:
   - `DATABASE_URL`: *Your Supabase Connection Pooler URL*
   - `DIRECT_URL`: *Your Supabase Direct Connection URL*
   - `JWT_SECRET`: *A secure, random 64-character hex string*
   - `NEXT_PUBLIC_APP_URL`: *Your Vercel deployment URL*
   - `CLOUDINARY_CLOUD_NAME`: *Cloudinary Cloud Name*
   - `CLOUDINARY_API_KEY`: *Cloudinary API Key*
   - `CLOUDINARY_API_SECRET`: *Cloudinary API Secret*
5. Click **Deploy**. Vercel will build the frontend pages, compile serverless routes, and optimize build traces.

---

## 5. Maintenance Commands

If the Prisma schema evolves in the future, run the following commands to sync your production Supabase database:

```bash
# Run migrations using the DIRECT_URL environment variable
npx prisma db push
```
